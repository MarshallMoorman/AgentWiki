- Modify AgentWiki-Project-Specification.md when requirements, architecture goals, or product-level behavior changes.
- Update docs/HANDOFF.md to reflect the current project state, outstanding work, and transfer guidance.
- Maintain docs/development/01-agents-readme-generation-requirements.md when README generation requirements or development processes change.
- Revise docs/plans/ui-companion-avalonia.md when the documented UI companion plan, assumptions, or implementation direction changes.
- Ensure documentation remains consistent across all referenced files and reflects the current repository state.

## Gotchas

- These files act as project guidance and specification sources; changes can affect developer and AI-agent behavior even without code changes.
- Keep terminology, requirements, and workflows synchronized across documents to avoid conflicting instructions.
- Document the current state of the project rather than speculative implementation details unless the file is explicitly a plan document.
- When changing requirements or architecture guidance, review all related documentation files for consistency.
